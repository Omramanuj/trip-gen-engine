Scenario~4 MIN READ
The checkout that buckled on launch day

A mid-size retailer is preparing for its largest sale of the year. Last season the checkout service held steady until 6pm, then latency climbed sharply and a third of payment requests timed out. Post-mortem traces pointed at a single Postgres primary serving both reads and writes, and a payment provider whose webhook retries amplified load during the incident.

The team has six weeks. They want to survive roughly ten times last year's peak without rewriting the whole platform, and they're willing to accept eventual consistency on anything that isn't money movement.

Your remit: across the next questions, you'll reason about the bottlenecks, propose a staged plan, and defend one trade-off out loud.

Assume a typical cloud environment, a small but capable team, and that observability is already in place. There is no single correct architecture — we're assessing how you reason, not whether you recall a pattern.

New components introduced

Everything above is built from v1 tokens plus these assessment-specific parts. Each is a standalone, reusable piece.

Screen frame

Top chrome (exit · context · autosave/timer), optional progress strip, body, and footer action bar. The container for every assessment moment.

layout
Progress strip

Segmented legs — done (teal), current (coral), upcoming (cream). Reading counts as leg zero.

navigation
Option card

MCQ answer with letter key, default / hover / selected states. Coral selection, never ambiguous.

input
Voice recorder

Record / stop, live waveform, timer, playback bar, and transcript preview. Re-record before submit.

input
Reading pack

Editorial passage layout with scenario callout and a persistent "re-open passage" affordance.

content
Dropzone

Drag-or-browse with format/size hints. Skins to video (sky) or PDF (coral) by context.

input
File card

Type chip, name, metadata, upload progress, and a status pill (uploading / ready / valid / in review).

feedback
Video player / recorder

16:9 frame with scrubber, plus a REC overlay state for in-browser screen capture.

media
Checklist

Self-verification list with done / to-do states. Used for rubrics and requirements.

content
Deadline banner

Amber async-task banner that replaces the timer for take-home trips.

feedback
Stamp / completion

Calm end state with the passport stamp, evidence tiles, and a status pill — never an undefendable score.

feedback
Guide line
